## 6. 当前已定位问题汇总

1. `model.cpp`
   - 旧实现按 `aiTextureType` 顺序顺推贴图，和 shader 固定槽位假设不一致，容易导致 albedo/metal-roughness 绑错，直接造成物体发黑或光照异常。

2. `model.cpp`
   - 旧实现无条件访问 `mesh->mTangents` 与 `mesh->mBitangents`，在缺失切线数据的模型上可能崩溃。

3. `model.cpp`
   - 旧实现没有检查 `Assimp::Importer::ReadFile` 返回值，模型加载失败时会直接解引用空场景。

4. `renderManager.cpp`
   - 旧实现上传点光源时硬编码 `intensity = 1.0` 和 `range = 65.0`，忽略场景 JSON 中配置的 `strength` 与 `zFar`，导致灯光表现与配置不一致。

5. `mesh.cpp`
   - 旧实现默认绑定 `textures[0..4]`，当某些材质没有贴图时会绑定到 0 号纹理对象，容易导致采样结果异常。
